# 보안·배포 가이드

본 프로젝트는 기본적으로 **동일 PC에서 클라이언트(WPF)·에이전트(FastAPI)가 `127.0.0.1`로 통신**한다는 전제입니다. 원격 또는 사내 배포 시 아래를 검토하세요.

## 위협 모델 요약

| 구분 | 로컬 개발(기본) | 원격/공유 네트워크 |
|------|----------------|-------------------|
| 전송 구간 | HTTP 평문, 루프백만 노출 | **반드시 TLS(HTTPS)** · 방화벽으로 포트 제한 |
| 인증 | 없음 | API 키·OAuth 등 애플리케이션 계층 인증 필요 |
| 저장 데이터 | `state.json`, 로그가 사용자 프로필/실행 폴더에 저장 | 디스크 암호화·백업 정책 정비 |

## 에이전트 서버(Python)

- **환경 변수**: `AGENT_HOST`, `AGENT_PORT`(기본 `8787`), 응답 접두사 `AGENT_REPLY_PREFIX`.
- 프로덕션에서는 **비밀키·API 키를 코드에 넣지 말고** 환경 변수 또는 비밀 저장소(OS Secret Manager, 컨테이너 시크릿 등)에서 주입합니다.
- 외부에 바인딩할 경우 **`127.0.0.1`가 아닌 `0.0.0.0`** 사용 시 방화벽 규칙과 인증을 반드시 병행합니다.

## 클라이언트(WPF)

- 현재 에이전트 URL은 코드에 고정되어 있습니다. 다른 호스트를 쓰려면 빌드 설정 또는 설정 파일 분리 후 배포 파이프라인에서 치환하는 방식을 권장합니다.
- 배포물에 **디버그 심볼·테스트 서버 주소**가 포함되지 않았는지 릴리스 전 확인합니다.

## 배포 체크리스트

1. 릴리스 빌드(`Release`)로 패키징, 불필요한 로그 레벨 비활성화.
2. 설치 프로그램 사용 시 **코드 서명**(가능한 경우).
3. 사내 배포: 엔드포인트를 사내 전용 네트워크/VPN 뒤에 두기.
4. 채팅·로그에 **개인정보·자격증명**이 남지 않도록 보존 기간·삭제 UX 정책 수립.
